Olink Functional Analysis (Pathway Enrichment) Guide

Functional Analysis with Olink

The functional analysis of Olink performs gene set enrichment analysis using MSigDB, Reactome, KEGG, or GO.
Two complementary approaches were applied: Gene Set Enrichment Analysis (GSEA) and Over-Representation Analysis (ORA).

About the Methods

  • GSEA (Gene Set Enrichment Analysis) :
    This method ranks all measured proteins based on their effect size (e.g., fold-change) and identifies pathways that are enriched toward the top or bottom of the ranked list.
  • ORA (Over-Representation Analysis) :
    This approach focuses on a set of proteins that passed a significance threshold (e.g., adjusted p-value) and evaluates whether any biological pathways are statistically over-represented in that set.

Important Considerations

  • The Olink panels are designed to target specific biological themes (e.g., inflammation, cardiometabolic processes), and do not cover the full human proteome.
    As such, pathway interpretation should always be made in the context of the proteins that were measured.
  • For example, when only the Inflammation panel is used, results will be biased toward immune-related pathways.
    This is expected and helps narrow biological interpretation to the relevant context.

Recommended Usage

  • Results are most robust when using broad protein coverage, such as the Olink Explore platform or multiple Target 96 panels.
  • For smaller protein lists, manual biological interpretation (e.g., keyword-based annotation) may provide more targeted insights than statistical enrichment alone.

Result Files

  • The results of the functional analysis are summarized in the Excel file named [Comparison]_enrichment_analysis.xlsx,
    which includes two sheets: gsea_results and ora_results.
  • For both GSEA and ORA, we have visualized the top 30 enriched functional terms using bar plots and heatmaps to provide a clearer overview of the most relevant pathways identified in the analysis.
    ([ORA|GSEA]_[Comparison]_heatmap.png and [ORA|GSEA]_[Comparison]_barplot.png)
  • Column information of each sheet is summarized in the table below.
  • Columns for ORA_results include:
    • ColumnDescription
    IDPathway ID from MSigDB
    DescriptionDescription of Pathway from MSigDB
    GeneRatioRatio of input proteins in the pathway
    BgRatioRatio of background genes in the pathway
    pvalueRaw enrichment p-value
    p.adjustAdjusted p-value (Benjamini-Hochberg)
    qvalueFalse discovery rate, the estimated probability that the normalized enrichment score represents a false positive finding
    geneIDList of enriched genes (Gene Symbols) annotated in a term delimited by "/"
    CountNumber of input proteins that are annotated in a term
  • Columns for GSEA_results include:
    • ColumnDescription
    IDPathway ID from MSigDB
    DescriptionDescription of Pathway from MSigDB
    setSizeNumber of input proteins that are annotated in a term
    enrichmentScoreEnrichment score, degree to which a gene set is over-represented at the top or bottom of the ranked list of genes
    NESNormalized Enrichment Score, normalized to account for differences in gene set size and in correlations between gene sets and expression data sets. NES can be used to compare analysis results across gene sets.
    pvalueRaw enrichment p-value
    p.adjustAdjusted p-value (Benjamini-Hochberg)
    qvalueFalse discovery rate, the estimated probability that the normalized enrichment score represents a false positive finding
    rankthe position in the ranked list where the maximum enrichment score occurred
    leading_edgeTags, list, and signal strength; Tags gives an indication of the percentage of genes contributing to the enrichment score. List gives an indication of where in the list the enrichment score is obtained. Signal represents the enrichment signal strength and combines the tag and list.
    core_enrichmentCore enriched proteins (Gene Symbols) annotated in a term delimited by "/"
